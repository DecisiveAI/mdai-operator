# SpanMetrics Observers

This document explains how to configure `spanMetrics` observers for the two supported providers:

1. `otelCollector` (OpenTelemetry Collector pipeline with `groupbyattrs` + `spanmetricsconnector`)
2. `greptimeFlow` (GreptimeDB flow/sink SQL generated by this operator)

Examples are based on `spanmetrics_observer.yaml`.

---

## Provider: `otelCollector`

### How the fields map

`spanMetricsObserver.otel.groupByAttrs`  
Attributes you want to **promote to resource level** before the spanmetrics connector runs. These are configured into the `groupbyattrs` processor as its `keys` list. The group-by-attributes processor groups spans (or logs/metrics) into resources based on the provided keys, effectively re-associating matching attributes at the resource level. `spanMetricsObserver.otel.connectorConfig`  
Raw `spanmetricsconnector` config (passed through and validated). The connector is listed as a contrib connector in the OpenTelemetry Collector, and configuration details are documented in the package docs. ŅłĆciteŅłéturn0search0Ņłéturn0search3Ņłü

### Example (from `spanmetrics_observer.yaml`)

```yaml
- name: watcher_gs_otel
  provider: otelCollector
  type: spanMetrics
  spanMetricsObserver:
    otel:
      # goes to groupbyattrs processor (keys)
      groupByAttrs:
        - some.field
      connectorConfig:
        dimensions:
          - name: host.name
          - name: server.address
```

### Important rules

1. **No overlap between `groupByAttrs` and `connectorConfig.dimensions`**  
   The webhook rejects duplicates because the same attribute would be used both as a resource grouping key and a spanmetrics dimension.

2. **`groupByAttrs` is optional**  
   If empty, the `groupbyattrs` processor is not inserted into the pipeline.

## Provider: `greptimeFlow`

### How the fields map

`spanMetricsObserver.greptime.dimensions`  
Dimension columns used in GreptimeDB sink tables and the `GROUP BY` clause in the generated flow SQL.

`spanMetricsObserver.greptime.primaryKey`  
Primary key column used in GreptimeDB sink tables.

### Example (from `spanmetrics_observer.yaml`)

```yaml
- name: watcher_gs_greptime
  provider: greptimeFlow
  type: spanMetrics
  spanMetricsObserver:
    greptime:
      # goes to GreptimeDB Flow query: tag columns and `group by` clause
      # need to be in format of sink table column names
      dimensions:
        - service_name
        - span_attributes.server.address
        - resource_attributes.host.name
      primaryKey: service_name
```

### What the operator generates (GreptimeDB)

The operator creates **three pipelines** (`traffic`, `latency`, `errors`). Each pipeline creates (or recreates) a sink table and a flow. Below are the exact templates (from `internal/controller/observer_greptime.go`), with your `dimensions` and `primaryKey` substituted at runtime:

**Sink table template (traffic; similar for latency/errors):**
```sql
CREATE TABLE IF NOT EXISTS golden_signals_traffic (
  "<dimension_1>" STRING INVERTED INDEX,
  "<dimension_2>" STRING INVERTED INDEX,
  ...
  total_count INT64,
  time_window TIMESTAMP TIME INDEX,
  update_at TIMESTAMP,
  PRIMARY KEY (<primaryKey>)
);
```

**Flow template (traffic; similar for latency/errors):**
```sql
CREATE OR REPLACE FLOW golden_signals_traffic_flow
SINK TO golden_signals_traffic
AS
SELECT
  "<dimension_1>" AS "<dimension_1>",
  "<dimension_2>" AS "<dimension_2>",
  ...
  COUNT(<primaryKey>) AS total_count,
  date_bin('5 seconds'::INTERVAL, timestamp) AS time_window
FROM opentelemetry_traces
WHERE span_status_code = 'STATUS_CODE_UNSET'
GROUP BY
  "<dimension_1>",
  "<dimension_2>",
  ...
  time_window;
```

**Latency flow** uses:
```sql
uddsketch_state(128, 0.01, duration_nano)
```

**Errors flow** uses:
```sql
WHERE span_status_code = 'STATUS_CODE_ERROR'
```

### Validation rules (Greptime)

- `dimensions` must be non-empty.
- `primaryKey` must be set.
- Names must match the identifier patterns enforced in `observer_greptime.go`.

---

## Quick checklist

- `otelCollector`: set `spanMetricsObserver.otel`, keep `groupByAttrs` and `connectorConfig.dimensions` **disjoint**.
- `greptimeFlow`: set `spanMetricsObserver.greptime.dimensions` and `primaryKey`.
